# MBOM 실험 환경 설명

MBOM 논문에서 사용하는 4개 multi-agent 환경에 대한 설명.

---

## 1. Triangle Game (삼각형 게임)

```
        L1
       /  \
      /    \
    L2 ---- L3

  P1 ●        ● P2
```

- **유형**: 비대칭 제로섬 게임
- **세팅**: 1×1 연속 공간에 2명의 플레이어 + 3개 랜드마크 (정삼각형 꼭짓점, 변 길이 0.6)
- **관측**: 14차원 — 자기 속도(2) + 자기 위치(2) + 랜드마크 상대좌표(6) + 상대 상대좌표(2) + 상대 속도(2)
- **행동**: 5개 이산 행동 (상/하/좌/우/정지), 물리 기반 힘 적용
- **에피소드**: 100 steps

### 규칙

각 플레이어는 랜드마크에 닿으면 (거리 < 0.15) 해당 랜드마크 상태가 됨. 매 step마다 두 플레이어의 랜드마크 상태 조합에 따라 아래 payoff matrix로 보상 결정:

|  | P2: Free | P2: L1 | P2: L2 | P2: L3 |
|---|---|---|---|---|
| **P1: Free** | 0, 0 | -0.5, +0.5 | -0.5, +0.5 | -0.5, +0.5 |
| **P1: L1** | +0.5, -0.5 | +1, -1 | +1, -1 | -1, +1 |
| **P1: L2** | +0.5, -0.5 | -1, +1 | +1, -1 | +1, -1 |
| **P1: L3** | +0.5, -0.5 | +1, -1 | -1, +1 | +1, -1 |

### MBOM 역할

P2 (MBOM)의 최적 전략이 P1의 상태에 완전히 의존. P1이 L1에 있으면 P2는 L3에 가야 하고, P1이 L2에 있으면 P2는 L1에 가야 함. 상대 전략 파악이 필수인 환경.

### 물리 파라미터

- 감쇠(damping): 0.25
- 시간 간격(dt): 0.1
- 최대 속도: 1.0
- 접촉 거리: 0.15

## 2. Coin Game (코인 게임)

```
┌───┬───┬───┐
│ R │   │   │
├───┼───┼───┤
│   │ 🪙│   │
├───┼───┼───┤
│   │   │ B │
└───┴───┴───┘
R = 빨강 에이전트, B = 파랑 에이전트, 🪙 = 코인 (색상 랜덤)
```

- **유형**: 반복 죄수의 딜레마 (고차원 확장)
- **세팅**: 3×3 토로이달 그리드 (양쪽 끝 연결), 2명 (빨강 agent 0, 파랑 agent 1)
- **관측**: 36차원 — 4채널 × 3×3 그리드 (자기 위치, 상대 위치, 빨강 코인, 파랑 코인)
- **행동**: 4개 이산 행동 (상/하/좌/우)
- **에피소드**: 150 steps

### 규칙

- 그리드에 코인 1개가 존재 (50% 확률로 빨강 또는 파랑)
- 코인 위치로 이동하면 줍기 성공
- 둘이 동시에 도착하면 agent 0이 우선

### 보상 구조

| 상황 | 줍는 사람 | 상대 |
|------|---------|------|
| 자기 색 코인 줍기 | **+1** | 0 |
| 상대 색 코인 줍기 | **+1** | **-2** |

### MBOM 역할

자기 색만 주우면 서로 손해가 없지만 (파레토 최적), 상대 색 코인을 빼앗으면 상대에게 -2 페널티. 이기적 전략과 협력 전략 사이의 균형을 학습해야 함. 논문에서는 **협력 환경**으로 분류.

## 3. Predator-Prey (포식자-피식자)

```
■ 장애물          P2 포식자(휴리스틱)
      P3 포식자(휴리스틱)

● 피식자(MBOM)       P1 포식자(학습)

      ■ 장애물
```

- **유형**: 비대칭, 다중 상대
- **세팅**: 1×1 연속 공간, 피식자 1마리 + 포식자 3마리 + 장애물 2개
- **관측**: 16차원 (피식자: 자기 속도/위치 + 장애물 + 포식자 위치, 포식자: + 피식자 속도)
- **행동**: 5개 이산 행동 (상/하/좌/우/정지)
- **에피소드**: 200 steps

### 규칙

- **피식자가 더 빠름**: 속도 1.3 vs 포식자 1.0
- 포식자가 피식자에게 충분히 가까워지면 (거리 < 0.075) 접촉
- 접촉 시: 피식자 **-10**, 포식자 **+10**
- 포식자 3마리 중 1마리만 학습 에이전트, 나머지 2마리는 휴리스틱 (단순 추격)

### MBOM 역할

MBOM이 피식자를 제어. 3마리 포식자를 하나의 "합동 상대"(joint opponent)로 모델링해야 함. 피식자가 더 빠르지만, 포식자들이 협동하면 포위 가능 → 상대 협동 패턴 예측이 핵심.

### 물리 파라미터

- 감쇠: 0.25, dt: 0.1
- 피식자 최대 속도: 1.3
- 포식자 최대 속도: 1.0
- 장애물 위치: (-0.3, 0.3), (0.3, -0.3)

## 4. Football Penalty (축구 페널티킥)

```
        ┌─────────────────┐
        │      골 대       │
        │    GK (MBOM)    │
        │                 │
        │       ⚽        │
        │    Shooter      │
        └─────────────────┘
```

- **유형**: 비대칭 제로섬, 고차원
- **세팅**: Google Research Football 기반 페널티킥 시나리오
- **관측**: 24차원 — 공 위치/방향(6) + 아군 위치/방향(8) + 적군 위치/방향(8) + 게임 모드(1) + 공 소유(1)
- **행동**: 11개 이산 행동 (8방향 이동 + 정지 + 슛 + 스프린트)
- **에피소드**: 가변 (골 성공 또는 시간 초과 시 종료)

### 규칙

- 슈터(agent 0)가 공을 드리블/슛
- 골키퍼(agent 1, MBOM)가 막아야 함
- 골 성공: 슈터 +1, 골키퍼 -1
- 골 실패: 보상 0

### MBOM 역할

골키퍼가 슈터의 슛 방향을 **예측**해서 미리 움직여야 함. sparse reward (에피소드 끝에 1번만 보상) + 고차원 상태공간 → 가장 어려운 환경. 슈터 전략이 바뀔 때 빠르게 적응해야 함.

## 논문의 실험 구성

논문은 3가지 시나리오를 커버하도록 실험을 구성함 (Section 5):

| 섹션 | 시나리오 | 환경 |
|------|---------|------|
| 5.1 Two-Player Zero-Sum Tasks | 2인 경쟁 | Triangle Game, One-on-One (Football) |
| 5.2 Multiple Opponents | 다중 상대 | Predator-Prey |
| 5.3 Cooperative Task | 협력 | Coin Game |

각 환경에서 3종류의 상대 (Fixed Policy, Naive Learner, Reasoning Learner)에 대해 평가. MBOM은 4개 환경 모두에서 기존 baseline (LOLA-DiCE, Meta-PG, Meta-MAPG, PPO)을 상회하며, 특히 naive learner와 reasoning learner 상대에서 성능 향상이 두드러짐 (논문 Table 1, Figure 2 참조).
